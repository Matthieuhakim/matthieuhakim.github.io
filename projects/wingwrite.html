What It Did

Two modes, one goal.

📧
Cold Email Generator
Users input a recipient, context, and tone — Wing Write used the OpenAI API to generate a personalized, human-sounding cold email ready to send. No templates, no copy-paste. Every email was different.
✍️
General Writing Assistant
Beyond cold email — a general AI writing assistant for drafting messages, professional intros, and outreach copy. Users could adjust tone, length, and style before finalizing.
Technical Build

Full-stack, zero shortcuts.

Wing Write was built entirely solo — frontend, backend, database, deployment, and AI integration. The stack was deliberately pragmatic: Java on the backend for reliability, JavaScript on the frontend for speed, SQL for data persistence.

The OpenAI API was called server-side to keep API keys off the client and give control over prompt engineering. Prompts were templated with user context and iterated based on output quality.

Reflection

What it taught me.

Wing Write was my first time building something end-to-end that real people actually used. Getting 1,000 users as a solo founder means you can't hide behind a team — every bug is yours, every user complaint is yours, and every moment of growth feels earned.

It also taught me the difference between shipping code and shipping products. The hardest parts weren't technical — they were figuring out what to build next and resisting the urge to add features instead of fixing friction.
