The Problem

Agents need API access.
But not this much.

Your AI agent needs a Stripe key to process refunds. But a raw API key means it could charge $1M or delete every customer. Today, developers either hand agents raw credentials and pray — or build bespoke guardrails from scratch for every deployment.

Bastion solves this at the infrastructure layer: agents get access to APIs without ever seeing the keys. Every call goes through a policy engine before it touches the real API. Every decision is cryptographically logged. Humans can be pulled in at the right moments.

How It Works

Five steps, every call.

01
Your agent sends a request to Bastion: "I want to charge $50 using credential X."
02
Policy check. Is this agent allowed to do this? At this amount? At this time of day? Against this IP? Bastion evaluates ABAC rules — allow, deny, or escalate.
03
On ALLOW: Bastion decrypts the credential from the vault, injects it into the outbound request, calls the external API, and returns the result. The key never leaves the server.
04
On ESCALATE: the request is held open while a human reviews it via the admin API. On approval the proxy flow resumes. On denial or timeout (5 min default), the agent gets a 403.
05
Every outcome — ALLOW, DENY, ESCALATE — is canonicalized, SHA-256 hashed, Ed25519-signed, and linked to the previous record. The chain is queryable and verifiable.
Core Features

Everything agents need.
Nothing they shouldn't have.

🔐
Credential Vault
Envelope encryption — AES-256-GCM + HKDF. Each credential gets its own data encryption key. Raw values are never returned over the API or stored in logs. Credentials can be scoped, revoked, or rotated without touching your agents.
⚖️
ABAC Policy Engine
Fine-grained rules per agent and credential. Wildcard action matching (charges.*), per-transaction and daily spend limits, Redis-backed rate limiting, timezone-aware time windows, IP allowlists, and approval thresholds that trigger ESCALATE.
🧑‍💻
Human-in-the-Loop Gate
When a policy triggers ESCALATE, the agent's request is held open while a human reviews. Admins approve or deny via the API or dashboard. Optional webhook notifications. The agent resumes transparently on approval — no retry logic needed.
🔗
Signed Audit Chain
Every ALLOW, DENY, and ESCALATE outcome is canonicalized, SHA-256 hashed, signed with the agent's Ed25519 key, and linked to the previous record. Query the chain at /v1/audit — or verify its integrity in one command.
🛡️
Fail Closed
No policy for an agent+credential pair means DENY. Bastion never silently allows a request. SSRF protection blocks localhost and cloud metadata endpoints out of the box. Security is the default, not an opt-in.
🔌
OpenClaw Plugin
Ships an OpenClaw plugin (@bastion-ai/bastion) that routes protected outbound tool calls through Bastion. Agents call bastion_fetch instead of direct tools — the plugin matches URLs to credentials and enforces policies transparently.

SDKs

Integrate in minutes.

Bastion ships first-class SDKs for TypeScript and Python. Zero runtime dependencies for the Node SDK.

TypeScript
Python
// npm install @bastion-ai/sdk import { BastionClient } from '@bastion-ai/sdk'; const client = new BastionClient({ baseUrl: 'http://localhost:3000', apiKey: agentSecret, }); const result = await client.execute({ credentialId: '<credential_id>', action: 'charges.create', params: { amount: 50 }, target: { url: 'https://api.stripe.com/v1/charges', method: 'POST', body: { amount: 5000, currency: 'usd' } }, }); // result.meta.policyDecision → "ALLOW" // result.upstream.body → Stripe's response
# pip install bastion-sdk from bastion_sdk import BastionClient with BastionClient("http://localhost:3000", api_key=agent_secret) as client: result = client.execute( credential_id="<credential_id>", action="charges.create", params={"amount": 50}, target={ "url": "https://api.stripe.com/v1/charges", "method": "POST", "body": {"amount": 5000, "currency": "usd"}, }, ) # Both sync and async clients available
What's Next

Bastion Cloud

Bastion is open source and self-hostable today. Bastion Cloud — the managed SaaS version — is in development. The vision: a hosted trust layer that any agent developer can plug into without running their own infrastructure.

If you're building with AI agents and want early access or want to contribute, the repo is open.
