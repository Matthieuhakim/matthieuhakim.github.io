DESA

Dynamic Emotional-State Architecture — replacing static persona prompts in LLMs with a continuously updating VAD emotional state vector, blended in real-time via LoRA adapters.

The Problem

Personas are frozen in time.

Current approaches to LLM personality and tone rely on static system prompts — a fixed description of a persona that doesn't change regardless of what happens in the conversation. A model told to be "warm and empathetic" stays that way whether the user just shared good news or is clearly frustrated.

Real emotional intelligence isn't static. DESA explores what happens when you give a language model a dynamic emotional state — one that updates continuously as the conversation evolves, and shapes the model's responses accordingly.

Old approach
Static Persona Prompt
"You are a warm, empathetic assistant." Fixed at session start. Unchanged regardless of user tone, topic shifts, or emotional escalation. The model performs a character — it doesn't inhabit one.
DESA approach
Dynamic VAD State Vector
A three-dimensional emotional state [V, A, D] updates after every exchange. LoRA adapters blended in proportion to this state shape the model's outputs in real-time. Emotional context accumulates — it isn't reset.
The Architecture

Three dimensions, infinite states.

DESA grounds emotional state in the Valence-Arousal-Dominance (VAD) framework from affective computing — a well-validated three-dimensional model of emotional space. Each dimension is a continuous value updated by a lightweight classifier after each conversational turn.

V
Valence
Positive vs. negative emotional tone. High valence = pleasant, enthusiastic. Low = frustrated, sad.
A
Arousal
Energy level. High arousal = excited, urgent. Low = calm, subdued. Shapes pacing and word choice.
D
Dominance
Sense of control or agency. High = assertive, confident. Low = uncertain, deferential.
LoRA Adapter Blending

State-conditioned model behavior.

The VAD state vector drives the selection and blending of LoRA (Low-Rank Adaptation) adapters — lightweight fine-tuned model components, each trained to produce outputs characteristic of a specific emotional region of the VAD space.

At inference time, adapters are blended in proportion to the current state vector. A shift in the user's tone doesn't require a new prompt — it nudges the blending weights, and the model's output style shifts accordingly. The emotional state is baked into the computation, not injected as text.

📐
Low-Rank Adaptation
LoRA fine-tunes only a small number of parameters — making it feasible to train multiple adapters per model without the cost of full fine-tuning for each emotional region.
🔀
Real-Time Blending
Adapters are mixed at inference time using the current VAD vector as blend weights. The model outputs emerge from a combination of emotional states, not a hard switch between them.
🔄
Continuous Update
A lightweight classifier updates the VAD vector after each conversational turn, tracking the emotional trajectory of the exchange over time — not just the last message.
🧪
PyTorch + HuggingFace
Implemented in PyTorch using HuggingFace Transformers and PEFT for LoRA adapter management. Experiments run on MIT's compute cluster.
